# Commit Changes

Commit the current changes to the codebase with a descriptive commit message.

## Flow

1. Check `git status --short` to understand the worktree.
2. Review the relevant diff before staging.
3. Stage only the files that belong in the commit.
4. Write a commit message that explains what changed and why.
5. Mention the issue or task identifier when one is known.

Use conventional commit message format if applicable.
